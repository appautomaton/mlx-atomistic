# DESIGN: GPCRmd Real Artifact MLX MD Run

## Problem

The active notebook should visualize a trajectory generated by `mlx_atomistic` from the downloaded GPCRmd 729 artifacts. Data availability and NBFIX are no longer the active blockers. The current blocker is large-system execution:

- GPCRmd 729 provides real CHARMM-style topology, coordinates, parameters, and protocol files.
- The PSF/PRM/PDB path now parses past derived MASS, Urey-Bradley, CMAP, and box extraction.
- CHARMM NBFIX type-pair overrides now export and load as compact prepared artifact arrays.
- NBFIX now applies inside `NonbondedPotential` as LJ sigma/epsilon substitution.
- A 92k-atom periodic GPCR membrane/solvent system currently fails because topology/nonbonded code tries to materialize dense nonbonded pairs.

## Data Flow

```text
GPCRmd cache 729
  -> atomistic_prep cache inspection
  -> CHARMM import normalization
  -> NBFIX type-pair override export
  -> strict PreparedSystem export
  -> mlx_atomistic artifact loader and nonbonded NBFIX runtime path
  -> lazy topology and scalable periodic nonbonded gate
  -> PME or explicit electrostatics blocker/prototype label
  -> short MLX NVT trajectory
  -> notebook PBC-aware visualization and analysis
```

Expected input roles:

- topology: `15286_dyn_729.psf`
- coordinates/model: `17686_dyn_729.pdb`
- parameters: `15290_prm_729.prm`
- protocol/start files: `17687_oth_729.tar.gz` / extracted `rep_*/input*`

Expected output bundle:

- `prepared_system.json`
- `prepared_system.npz`
- `view.pdb`
- `gpcrmd_import_report.json`
- `gpcrmd_mlx_run_report.json`
- `trajectory.npz` only when `mlx_atomistic` actually runs

## Import Gate

`atomistic_prep` owns conversion from GPCRmd files into a strict prepared artifact. It must not simulate.

Required behavior:

- Resolve the GPCRmd cache and role files deterministically.
- Extract or derive only information present in the GPCRmd files.
- Generate any PSF-derived CHARMM MASS prelude as a derived parser aid, not as invented parameters.
- Parse `input.xsc` or equivalent protocol files for periodic box vectors.
- Preserve required CHARMM terms in artifact arrays:
  - bonds, angles, proper torsions, impropers;
  - Urey-Bradley terms;
  - CMAP terms and grids;
  - nonbonded exceptions;
  - NBFIX atom-type pair override values.
- Fail closed if required terms or fields cannot be exported.

NBFIX is represented as a compact CHARMM atom-type-pair override table, not an O(N²) expanded pair list:

- atom type identifier pairs;
- override LJ sigma/Rmin-equivalent value after unit conversion;
- override epsilon after unit conversion;
- source counts and source atom type names for diagnostics.

If ParmEd exposes distinct 1-4 NBFIX values or malformed NBFIX entries that the runtime cannot represent, the importer must fail closed with a named blocker. It must not silently drop the override or approximate it with ordinary Lorentz-Berthelot mixing.

## Runtime Gate

`mlx_atomistic` owns trajectory generation. It must not call external MD engines.

Required behavior:

- Load only strict prepared artifacts.
- Reject unsupported artifact terms before integration starts.
- Apply NBFIX inside nonbonded LJ evaluation. Base nonbonded must be evaluated once; NBFIX changes LJ parameters for matching atom-type pairs and must not duplicate Coulomb or the full nonbonded term.
- Preserve topology exclusions and explicit nonbonded exceptions. NBFIX does not re-enable excluded pairs, and explicit exceptions remain authoritative for their listed pairs.
- Allow NBFIX with PME/Ewald Coulomb because NBFIX modifies LJ parameters, not Coulomb electrostatics.
- Avoid eager dense all-pairs matrices for 92k atoms at topology construction time.
- Use compact periodic neighbor-list nonbonded execution for full-system real-space cutoff work.
- Fail before integration if the selected nonbonded backend would require O(N²) storage or dense all-pairs temporaries for the full GPCRmd system.
- Treat PME as a separate electrostatics gate. The current standalone/reference PME implementation may be useful for correctness fixtures, but GPCRmd dynamics must either run through an executable MLX PME path or be clearly blocked/prototype-labeled before stepping.
- Run short NVT proof dynamics with conservative settings.
- Save finite diagnostics, sampled coordinates, velocities, box vectors, and metadata.

The first success target is a short proof trajectory, not long-timescale GPCR behavior. NPT and membrane barostat remain outside this change unless the proof trajectory cannot be honestly gated without them.

## Notebook Gate

`notebooks/ligand-receptor-motion/` is a consumer of MLX artifacts.

Required behavior:

- If `trajectory.npz` is missing, call or show `run-gpcrmd-mlx` status and display blocker JSON.
- If `trajectory.npz` exists and metadata says `engine=mlx_atomistic`, load it.
- Render degraded large-system defaults: receptor, ligand, selected pocket residues, nearby waters/ions, and box-aware path traces.
- Keep PBC-aware distance/contact analysis.
- Never show GPCRmd reference trajectory playback as the active result.
- Never label the short NVT proof as binding, unbinding, or production-timescale GPCR dynamics.

## Vendor Policy

`vendors/` may be read only for semantic reference, for example to understand CHARMM CMAP, PME, NBFIX, or neighbor-list behavior. Runtime code must not import from, link to, execute, or build against vendor checkouts.

## Failure Contract

Every blocked run should emit machine-readable blockers in `gpcrmd_mlx_run_report.json`. Acceptable blockers include:

- missing GPCRmd role files;
- parser missing or parser failure;
- unsupported CHARMM terms;
- missing box vectors;
- unsupported NBFIX semantics;
- dense nonbonded route rejected for full system;
- missing lazy topology support;
- missing scalable periodic neighbor-list nonbonded support;
- PME not executable for the selected GPCRmd proof mode;
- MLX runtime memory/performance failure with exact command and system size.

Fabricated trajectories, stripped toy systems, and silently dropped force-field terms are not acceptable outcomes.
