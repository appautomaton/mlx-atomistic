# PLAN: GPCRmd Real Artifact MLX MD Run

## Goal

Use the downloaded GPCRmd 729 artifacts to produce a strict MLX-ready prepared artifact, run a short real `mlx_atomistic` NVT trajectory, and make `notebooks/ligand-receptor-motion/` visualize/analyze that MLX-generated trajectory.

The workflow must either write a real `trajectory.npz` generated by `mlx_atomistic`, or return precise blockers. It must not play public trajectories, run external MD engines, drop required CHARMM terms, or fall back to toy systems.

## Architecture Approach

Use fail-closed gates before notebook visualization:

1. `atomistic_prep` converts GPCRmd PSF/PDB/PRM/protocol files into a strict prepared artifact with topology, parameters, masks, box vectors, constraints, and CHARMM terms preserved.
2. `atomistic_prep` exports NBFIX as executable CHARMM type-pair LJ override data, not as metadata-only rejection text.
3. `mlx_atomistic` applies NBFIX inside the nonbonded LJ path while preserving topology exclusions, explicit nonbonded exceptions, and PME/periodic Coulomb handling.
4. `mlx_atomistic` separates topology bookkeeping from executable nonbonded pairs so 92k-atom artifacts can load without O(N^2) pair materialization.
5. `mlx_atomistic` routes large periodic nonbonded work through a compact neighbor-list path and blocks before integration if the runtime would fall back to dense all-pairs work.
6. PME is treated as a GPCRmd-critical runtime gate: the current standalone/reference PME path is not enough for a real 92k-atom run unless the selected proof command explicitly uses a validated short-range prototype mode.
7. The notebook remains an artifact consumer. It visualizes only when metadata confirms the trajectory was generated by `mlx_atomistic`.
